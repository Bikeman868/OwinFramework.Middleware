Owin Framework form identification middleware

The form identification middleware in the Owin Framework supports the following configuration options:

Property Default Current Description
secureDomain {secureDomain.default} {secureDomain}

Specifies the domain name to use in URLs that need the user identification cookie. By using a separate domain for these requests, the main web site can be insecure and user identification cookie will not be leaked.

Leaving this blank or setting is to the same domain name as the site itself will also work, but in this case you must run all requests over HTTPS and never use HTTP.

sessionName {sessionName.default} {sessionName}

Specifies the name to use when saving the user identity token in the session store.

documentationPage {documentationPage.default} {documentationPage}

The relative path to this documentation page. Leave blank to disable documentation in production environments, or prevent this page from being retrieved from outside your local network.

signupPage {signupPage.default} {signupPage}

The relative path to POST to when users want to create a new account. Leave blank to prevent users from creating accounts.

signinPage {signinPage.default} {signinPage}

The relative path to POST to when users want to log in to their account. Leave blank to prevent users from logging in (for example if you only want to use the shared secret identification method).

signoutPage {signoutPage.default} {signoutPage}

The relative path to POST to when users want to log out from their account. Leave blank to prevent users from logging out (for example if you only want to use the shared secret identification method).

signupSuccessPage {signupSuccessPage.default} {signupSuccessPage}

Users will be redirected to this page after succesfully creating an account. Leave blank to leave the user on the page where they created their account.

signupFailPage {signupFailPage.default} {signupFailPage}

Users will be redirected to this page after failing to create an account. Leave blank to leave the user on the page where they tried to create an account.

signinSuccessPage {signinSuccessPage.default} {signinSuccessPage}

Users will be redirected to this page after successfully logging into an account Leave blank to leave the user on the page where they logged in.

signinFailPage {signinFailPage.default} {signinFailPage}

Users will be redirected to this page after failing to log into an account Leave blank to leave the user on the page where they tried to log in.

signoutSuccessPage {signoutSuccessPage.default} {signoutSuccessPage}

Users will be redirected to this page after logging out from their account. Leave blank to leave the user on the page where they logged out.

sendPasswordResetPage {sendPasswordResetPage.default} {sendPasswordResetPage}

Send a POST request to this page to trigger the sending of an email containing a time limited single use password reset token. The email will contain a link to the reset password page with the password reset token in the query string.

resetPasswordPage {resetPasswordPage.default} {resetPasswordPage}

Send a POST request to this page to reset a user's password. The form fields must contain a valid password reset token that has not been used before.

renewSessionPage {renewSessionPage.default} {renewSessionPage}

This middleware will redirect the browser to this page whenever the users session does not contain a user identity. This page will be handled by this middleware by identifying the user and putting their identity in session before redirecting back to the original page.

This middleware is configured the same way as all other middleware in the Owin Framework; you need to construct a class that implements IConfiguration then add a fluid .ConfigureWith() when you register this middleware with the Owin Framework builder. Please read Owin Framework documentation on configuration for full details and examples.
