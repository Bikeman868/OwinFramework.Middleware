Owin Framework session middleware configuration

The session middleware in the Owin Framework supports the following configuration options:

Property Default Current Description
Template {cacheCategory.default} {cacheCategory}

The session object will be stored in the ICache facility using this category name. The cache facility can be configured with different behaviours for each category of information stored in it.

Template {sessionDuration.default} {sessionDuration}

The amount of time that the session is valid for. After this time expires the session will be deleted and re-created empty. Downstream middleware treats the session as a cache and rebuilds missing information from backing store when the session expires.

Template {cookieName.default} {cookieName}

The name of the cookie to store on the browser that holds the session ID.

Template {cookieDomainName.default} {cookieDomainName}

The domain name to store the cookie under on the browser.

DocumentationRootUrl {documentationUrl.default} {documentationUrl}

The relative path to this documentation page.

This middleware is configured the same way as all other middleware in the Owin Framework; you need to construct a class that implements IConfiguration then add a fluid .ConfigureWith() when you register this middleware with the Owin Framework builder. Please read Owin Framework documentation on configuration for full details with examples.
