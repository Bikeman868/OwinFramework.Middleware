Owin Framework default document configuration

The default document middleware in the Owin Framework supports the following configuration options:

Property Default Current Description
Enabled {enabled.default} {enabled}

Set this to 'false' to disable the default document middleware

defaultPage {defaultPage.default} {defaultPage}

When a request is received for the root of the web site with no path, the request will be re-writen to this path and processed through the OWIN pipeline as if this has been the request that was received.

documentationRootUrl {configUrl.default} {configUrl}

The relative path to this documentation page. Set this to an empty string to disable the documentation endpoint.

paths {paths.default} {paths}

A list of additional sub-paths within the site where a default document is also required

This middleware is configured the same way as all other middleware in the Owin Framework; you need to construct a class that implements IConfiguration then add a fluid .ConfigureWith() when you register this middleware with the Owin Framework builder. Please read Owin Framework documentation on configuration for full details with examples.
