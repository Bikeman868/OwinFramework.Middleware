OwinFramework less middleware configuration

The less middleware in the OwinFramework support the following configuration options:

Property Default Current Description
RootUrl {rootUrl.default} {rootUrl}

Configures the URL within your web site where css files will be served. If you leave this property empty then the root folder for the web site will be the root url for css files.

For example setting this property to /styles means that any requests to your site that follow the pattern /styles/*.css will be served as css/less files.

This middleware will only serve requests for .css files. If the url maps to a physical file then this file will be served. If there is a .less file with the same name then it will be compiled on the fly into css and served.

DocumentationRootUrl {documentationRootUrl.default} {documentationRootUrl}

Configures the URL within your web site where configutation documentation for this middleware will be avialble. Set this to an empty string to disable this documentation in a production environment.

RootDirectory {rootDirectory.default} {rootDirectory}

Configures the path in the file system where css files reside. This path can be relative to the root folder of the site, or an absolute path to anywhere on the file system including a UNC path to a network resource.

Enabled {enabled.default} {enabled}

The Enabled property allows you to turn off the less file handler.
AnalyicsEnabled {analyicsEnabled.default} {analyicsEnabled}

Allows you to turn off the analytics for this middleware

This middleware is configured the same way as all other middleware in the Owin Framework; you need to construct a class that implements IConfiguration then add a fluid .ConfigureWith() when you register this middleware with the Owin Framework builder. Please read Owin Framework documentation on configuration for full details with examples.
